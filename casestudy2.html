Event Management System: Transforming Country Club Operations

A user-centered design solution that streamlines event check-ins and enhances staff efficiency.

Project Overview

Country clubs face operational challenges with manual event management processes that lead to member dissatisfaction and staff inefficiency. Starting with customer interviews to identify pain points before designing a comprehensive Event Management System (EMS).

As the lead UX designer and researcher, I drove the discovery process, synthesized findings into actionable insights, and created a high-fidelity prototype that addressed key usability issues and improved the check-in experience for staff.

Prototype: Hi-fi Prototype (Figma)

The Problem

Country club staff were managing events using fragmented manual processes (paper sign-in sheets, spreadsheets, and multiple disconnected tools). This led to:

  • Long check-in lines and member frustration
  • Inaccurate attendance records affecting catering and planning
  • Inefficient communication between departments (catering, events, membership)
  • Missed opportunities to personalize member experiences
  • 15+ hours weekly spent on manual data entry and reconciliation

Research & Discovery

We began with comprehensive research to understand staff needs and pain points.

Research Methods

  • 12 In-depth Interviews with staff across roles (General Manager, Membership Engagement, Catering, Event Coordination)
  • Contextual Inquiry observing actual event check-in processes
  • Competitive Analysis of existing event management platforms
  • Affinity Mapping to synthesize findings into actionable insights

Key Research Insights

  • Manual processes created errors: 100% of staff reported data entry mistakes from paper-based systems
  • Real-time information gap: No single source of truth for attendee counts and last-minute changes
  • Privacy concerns: Staff needed to limit visible personal data during check-in
  • Integration requirements: Any solution must work with existing member databases

Usability Testing & Iteration

Following the high-fidelity prototype, we conducted moderated usability tests with 5 participants representing our key user personas. I led the test with Ella, an event coordinator, to gather deep feedback on the check-in workflow.

The testing revealed critical insights that shaped our final design:

  • Users were confused, thinking both "Name" and "Member ID" were required for manual check-in
  • The failure state of the automated QR check-in was not prominent enough
  • Dashboard navigation needed to be more intuitive

Key Finding

Users assumed both "Name" and "Member ID" were required for manual check-in, causing hesitation and confusion during the process.

My Solution

Changed the label from "Name and MemberID" to "Full Name or MemberID" with a clear "OR" indicator between fields.

Key Finding

Users didn't always realize when automated check-in failed, making it unclear whether check-in was successful or an error occurred.

My Solution

Introduced a prominent error modal with a clear call-to-action, guiding users to manual check-in when automation failed.

Key Finding

Participants found event selection on the dashboard unclear and were uncertain about where to begin after logging in.

My Solution

Restructured the dashboard with a calendar-based navigation menu for quicker, more intuitive access to events.

Key Finding

There was no dedicated scan button for QR code check-ins, causing confusion about when the scanning process started.

My Solution

Added a dedicated "Scan" button and a greyed-out scanning button on the next screen to indicate successful scanning.

Final Design Solution

The EMS prototype includes four interconnected components that address the specific needs of each user persona:

1. Streamlined Check-In Process

  • One-tap check-in via QR code or membership card scan with instant confirmation
  • Staff-assisted name search for fallback scenarios
  • iPad kiosk mode for self-service check-in

2. Real-Time Staff Dashboard

  • Live attendee counts with color-coded capacity indicators
  • Dietary restriction alerts for catering staff
  • Last-minute cancellation notifications
  • Quick export functionality for reports

3. Comprehensive Member Profiles

  • Event participation history and preferences
  • Dietary restrictions and special requirements
  • Family member connections and relationships

4. Reporting & Analytics Suite

  • Attendance trends and participation metrics
  • Event popularity analysis
  • Member engagement scoring
  • ROI calculation for event planning

Reflection & Learning

This project reinforced the importance of deep customer understanding before solutioning. By applying Amazon's working backwards approach, we ensured every design decision traced back to a validated user need.

Key Learnings

  • Fallback options are critical: Even in high-end environments, not all users are comfortable with technology
  • Data privacy concerns: Staff needed ways to protect member information during check-in processes
  • Integration requirements: New tools must work within existing tech ecosystems to be adopted
  • Value of rapid prototyping: Getting feedback early prevented expensive redesigns later
